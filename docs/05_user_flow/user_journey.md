# User Flow - App Ingreso

## Principios del flujo

- Baja fricción: el usuario empieza a practicar rápido
- Feedback inmediato en cada paso
- Enfoque en acción, no en lectura
- El progreso depende del uso
- El sistema guía, no enseña

---

## 1. Entrada al sistema (Onboarding)

### Objetivo
Que el usuario entienda qué es la app y empiece rápido.

### Flujo

1. Pantalla inicial
   - Qué hace la app
   - Mensaje claro: entrenamiento por práctica

2. Perfil básico
   - Nombre
   - Edad (opcional)

3. Explicación breve
   - “Vas a practicar ejercicios”
   - “El sistema te muestra en qué mejorar”

4. Inicio inmediato
   - Botón: "Empezar"

---

## 2. Diagnóstico inicial

### Objetivo
Detectar nivel sin fricción.

### Características

- corto (5–10 ejercicios)
- mezcla de habilidades
- sin explicación previa

### Resultado

- nivel inicial por habilidad
- no mostrar complejidad técnica al usuario

---

## 3. Entrenamiento diario

### Objetivo
Generar hábito de práctica.

### Flujo principal

1. El sistema propone ejercicios
2. El usuario responde
3. Feedback inmediato
4. Siguiente ejercicio

### Características

- sesiones cortas (5–10 min)
- ejercicios rápidos
- sin pantallas innecesarias

---

## 4. Sistema de feedback

### Correcto

- confirmación inmediata
- refuerzo positivo simple

### Incorrecto

- explicación breve
- mostrar lógica correcta
- continuar sin fricción

---

## 5. Adaptación del sistema

### Objetivo
Ajustar dificultad automáticamente.

### Reglas visibles para el usuario

- “seguí practicando esto”
- “vas mejorando”

### Reglas internas

- repetir habilidades con errores
- subir dificultad progresivamente

---

## 6. Simulador

### Objetivo
Reproducir experiencia de examen.

### Características

- tiempo limitado
- mezcla de habilidades
- sin ayuda

### Resultado

- puntaje
- resumen por habilidad

---

## 7. Progreso del alumno

### Qué se muestra

- nivel por habilidad
- porcentaje de aciertos
- evolución

### Qué NO se muestra

- métricas complejas
- datos técnicos

---

## 8. Intervención del padre

### Rol del padre

- controlar uso
- ver progreso

### Qué ve

- tiempo de uso
- avance general
- consistencia

### Qué NO hace la app

- no reemplaza seguimiento
- no garantiza resultados

---

## 9. Retención

### Estrategias

- sesiones cortas
- progreso visible
- refuerzo constante

---

## 10. Loop principal del producto

Entrar → Practicar → Feedback → Repetir → Simular → Ver progreso → Volver a practicar

---

## Nota

El flujo debe ser simple, repetitivo y claro.

El objetivo no es enseñar, sino lograr que el usuario practique lo suficiente para mejorar.
